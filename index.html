3. WBS 소요기간 산정식 (작업일→달력일)

입력된 “작업일(Workdays)”은 비작업일(연간)과 현장난이도 배수를 반영해 “달력일(Calendar days)”로 변환된다.

r = N_nonwork / 365
CF = 1 / (1 - r)

WBS 구성(요약)
  • - 가설공사(현장설비/동선/가설전기) → 흙막이·토공·기초(직렬)
  • - 파일공사(옵션): 가설 완료 후, 토공 착수 후 SS-lag로 부분 착수 허용
  • - 지하 골조(옵션): 토공 및 파일 완료 후 진행
  • - 지상 골조: F1…F(n) 반복(지상층수+옥탑 1개층)
  • - 외장/벽체/MEP/내부마감: 층별 반복 + SS/FS 연계
  • - 전기·통신·소방(별도): 비용 0으로 표시(일정 CP에는 영향 가능)
  • - TAB·시운전·성능검증 → 준공/검사/인허가

4. 네트워크(선후관계) 및 CPM 산출식

본 엔진은 활동들을 노드로, 선후관계를 간선(edge)로 두는 네트워크를 구성하고, Forward/Backward pass로 ES/EF/LS/LF 및 Float를 계산한다. (FS/SS + lag 지원)
Forward pass (Earliest)
ES_i = max( ES_j + Lag_SS(j→i), EF_j + Lag_FS(j→i) )
EF_i = ES_i + d_i

구현에서는 “가중치 w(j→i)”로 통일: FS는 w=d_j+lag, SS는 w=lag.

Backward pass (Latest)
LS_i = min( LS_k - w(i→k) ) for all successors k
LF_i = LS_i + d_i
Float_i = LS_i - ES_i
CP ⇔ Float_i ≈ 0
CP의 “동적 변화” 처리
  • - 슬라이더/목표단축 적용 시 각 활동의 d_i가 변함 → CPM 재계산 → Float 재산정
  • - 기존 Non-CP였던 공정이 Float 소진으로 CP로 전이될 수 있음
  • - 복수 CP가 동시에 존재 가능 → 단축 시 “모든 CP를 동시에 줄이는” 조합 선택이 필요

5. 단축비용 산정식 (Direct + Indirect)

활동별 단축비용 기울기(Cost Slope)
s_i = (C_i(crash) - C_i(normal)) / (d_i(normal) - d_i(crash))
d_i ∈ [d_i(crash), d_i(normal)]

전기·통신·소방(별도)는 비용 0(표시)로 두되, 일정 CP에는 영향을 줄 수 있으므로 일정 계산에는 포함된다.

직접비 누계(추가분)
C_dir = C_base + Σ (d_i^N - d_i) · s_i
간접비(일일 유지비)
C_ind = D · c_ind/day
C_total = C_dir + C_ind

c_ind/day는 (기본) 비율 기반 자동 또는 사용자 입력값(override)을 사용.

6. 복수 CP 단축(최소컷) 알고리즘

단축을 1일 수행할 때 CP가 하나면 “가장 싼 CP 공정”만 줄이면 되지만, 복수 CP가 동시에 존재하면 한 경로만 줄여서는 프로젝트 기간이 줄지 않을 수 있다. 따라서 CP 서브그래프에서 최소 비용 컷(min-cut)을 찾아, “모든 CP를 동시에 단축시키는” 공정 집합을 선택한다.
(개념) CP 상의 타이트(edge)들만 남긴 그래프 Gc 구성
각 활동 i에 노드 용량 cap(i)=s_i 부여(단축 가능 시)
s→(CP 시작노드), (CP 종료노드)→t 연결 후, s-t 최소컷 = 최소 단축비 조합
선택된 활동들의 d_i를 1일 감소 → CPM 재계산 → 반복
※ 구현은 Dinic MaxFlow로 min-cut을 구해 “1일 단축 시 필요한 최소 비용 조합”을 반복 적용하며, 매 단계의 (기간, 비용) 포인트를 저장해 U-Curve를 생성한다.
